ROOT CAUSE ANALYSIS
═══════════════════════════════════════════════════════

ROOT CAUSE 1 (Primary — why live site showed minimal billing dashboard):
  File: landing/_redirects  AND  landing/netlify.toml
  Issue: /dashboard was routed to /dashboard.html (static minimal billing HTML), not the React SPA.
         /login was routed to /login.html (HTML form), not React LoginPage.
         netlify.toml [build] had no build command, never built the React SPA.
  Fix: Rewrote _redirects to route all app paths to /app/index.html (React SPA).
       Updated netlify.toml to build the dashboard into landing/app/.
       Deleted landing/dashboard.html and landing/login.html (static file override blocked redirects).

ROOT CAUSE 2 (Router mismatch — login would fail after redirect):
  File: apps/web-dashboard/src/pages/LoginPage.tsx line 2
  Issue: `useLocation` imported from 'wouter' — a package incompatible with the react-router-dom
         router used everywhere else in App.tsx; navigate() would not fire React Router navigation.
  Fix: Replaced `useLocation` from 'wouter' with `useNavigate` from 'react-router-dom'.
       Changed navigate('/') → navigate('/dashboard', { replace: true }).

ROOT CAUSE 3 (Build config — SPA was never emitted to landing/):
  File: apps/web-dashboard/vite.config.ts
  Issue: outDir was 'dist' (per-app local), base was '/' (wrong for /app/ subpath).
         'wouter' was in manualChunks causing unnecessary coupling.
  Fix: outDir → '../../landing/app', base → '/app/', removed 'wouter' from chunks.

ROOT CAUSE 4 (Auth token mismatch):
  File: landing/login.html
  Issue: Stored token as 'zonforge_access_token' but React SPA reads 'zf_access_token'.
         Users who logged in via HTML form appeared logged out in the React app.
  Fix: Removed login.html — /login now serves React LoginPage which stores correct key.

ROOT CAUSE 5 (API proxy missing):
  File: landing/_redirects
  Issue: SPA pages with hardcoded fetch('/api/...') paths would 404 or get caught by wildcard.
  Fix: Added /api/*  →  https://api.zonforge.com/:splat  200  BEFORE the wildcard rule.
