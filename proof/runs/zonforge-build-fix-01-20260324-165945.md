# ZONFORGE-BUILD-FIX-01

## 1. Initial Failure Summary
Deployment builds were failing in Docker build stage around root Dockerfile COPY steps that expected pnpm workspace artifacts. Failure class matched Dockerfile assumptions not aligned with actual repo package manager/workspace files.

## 2. Exact Root Cause
**Class G — Wrong package manager assumption** (with lockfile mismatch side effect).

The root Dockerfile was written for a pnpm workspace and explicitly copied and used:
- `pnpm-lock.yaml`
- `pnpm-workspace.yaml`
- `pnpm install --frozen-lockfile`

But repository root is an **npm workspace** (`packageManager: npm@10.5.0`) and contains `package-lock.json` with no pnpm lock/workspace files.

## 4. Why This Fix Is Correct
Minimal non-breaking Dockerfile patch only:
- Replaced pnpm workspace copy inputs with npm workspace files (`package-lock.json`, `tsconfig.base.json`).
- Replaced pnpm install/build commands with npm workspace equivalents.
- Removed pnpm corepack activation lines.

No UI, business logic, or product behavior was changed.
